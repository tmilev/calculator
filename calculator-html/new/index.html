The calculator is a standalone application that can either be used as a web server (similarly to SAGE) or via an Apache web server. The standard way of running the calculator is as a stand-alone linux webserver. If you want to run the calculator through Apache, best is to contact the authors for instructions. To get the calculator as a stand-alone linux webserver do the following.
0) You need a Linux machine. Tested on Ubuntu, OpenSUSE and CENTOS. 1) Install subversion. On Ubuntu the command is as follows.
sudo apt-get install subversion
2) Install g++ (the minimum for compiling c++ programs). On Ubuntu the command is the following.
sudo apt-get install g++
3) [Optional] Install the development packages for mysql and openssl. To do so, google-search ``how to install ... on ...''"
4) Checkout the calculator project. The command is as follows.
svn checkout https://github.com/tmilev/calculator.git/trunk calculator
The command fetches the latest source code from sourceforge.
5) Navigate to the newly created calculator folder.
6) Type the following command.
make nossl=1 nosql=1
Alternatively, if you installed mysql and openssl in the preceding optional step, you can use the following command.
make
Wait until the command is completed (takes about 10 minutes on my machine when compiled using one core). The installation is now complete. To run the calculator, see the next two points.
7) Through the command-line, navigate to the following directory.
calculator/bin/
8) Type the following command. Make sure to include the dot.
./calculator server
9) The calculator will display a message saying on which port it got bound. The default ports should be 8155 and 8166 for http and https. Type the address:
http://localhost:8155/cgi-bin/calculator
in your web browser to get to the calculator. If the calculator didn't succeed in binding to port 8155 (say, the port is already taken) it will try to bind next to ports 8080, 8081 and 8082. In that case replace the 8155 in the address above with the port number reported by the calculator.
